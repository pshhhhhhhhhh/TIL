# 디자인패턴 (Gang of Four)
- 디자인패턴은 자주 발생하는 문제를 해결하기 위한 반복적 해결 방법임
- 생성, 구조, 행위 패턴이 있음
- 생성 패턴 *팩토리나 빌더가 나오면 일단 생성으로 박음*

    - 빌더
    - 팩토리 메서드
    - 프토토타입
    - 싱글톤 
- 구조 패턴
    - 어댑터
    - 브릿지
    - 컴포지트
    - 데코레이터
    - 퍼사드
    - 프라이웨잇
    - 프록시
- 행위 패턴
    - 메멘토
    - 옵저버
    - 상태

    - 템플릿
